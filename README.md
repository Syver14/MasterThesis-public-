# MGS-1 Reduced-Mass Geotechnical Index Testing

This repository contains the data, tables, calculation notebook, and thesis figure export workflow for reduced-specimen-mass geotechnical index testing of MGS-1 Mars regolith simulant.

## Contents

- `MGS1_analysis_github_final_figures.ipynb` - recommended GitHub notebook. It reproduces the final selected thesis figure set only.
- `MGS1_analysis_github_reproducible.ipynb` - full reproducible workflow. It keeps all generated figures, including alternative/review versions.
- `MGS1_analysis_rev03_word_export.ipynb` - working thesis export notebook kept as the source revision behind the GitHub notebook.
- `data/` - cleaned input data and source workbooks.
- `tables/` - generated CSV tables used in the thesis.
- `figures/` - archival PDF figures generated by the notebook.
- `figures_word/svg/` - recommended Word-ready vector figures.
- `figures_word/png_450dpi/` - final-size high-resolution PNG fallback for Word.
- `figure_manifest.csv` - figure-number-to-file mapping for thesis insertion.
- `requirements.txt` - Python package requirements.

## Figure Export Notes

The notebook draws figures at the intended Word text width (`6.5 in`) instead of exporting very large PNG files and relying on Word to downscale them. This gives more predictable text size, line weight, and PDF export quality.

Use the SVG files in `figures_word/svg/` first. They are vector graphics and should stay sharp when Word exports the thesis to PDF. If a specific SVG causes layout problems in Word, use the matching PNG from `figures_word/png_450dpi/`.

The PNG fallback is exported at 450 dpi at final thesis width. This is intentionally smaller than the earlier 900 dpi export because oversized raster files can be resampled and softened by Word during sharing or PDF export.

## Reproducing the Outputs

From the repository root:

```bash
pip install -r requirements.txt
jupyter notebook MGS1_analysis_github_final_figures.ipynb
```

Run all cells in the notebook. It regenerates the thesis PDF figures, Word-ready SVG files, PNG fallbacks, and generated CSV tables.

Use `MGS1_analysis_github_reproducible.ipynb` instead if you want to regenerate the full analysis workflow, including alternative figure versions that are not retained in the final thesis figure set.

## Recommended GitHub Contents

Commit the reproducible notebook, `README.md`, `requirements.txt`, `figure_manifest.csv`, the CSV files in `data/`, and the generated `tables/` and final figure exports if you want readers to inspect the results without running the notebook.

Do not commit local review sheets, temporary figure-contact images, notebook checkpoints, or old backup export folders. The `.gitignore` file is set up to keep those out of the repository.
